## 📌 專案簡介 
近年新聞頻傳因無人看火」或瓦斯管線老舊洩漏導致的住宅意外，造成嚴重財損與傷亡。傳統廚房安防設備常依賴單一物理感測器（如煙霧、瓦斯警報器），在實際烹飪情境中極易因高溫油煙或水蒸氣引發誤報。

本專題之核心創意為 **「多模態資料融合判定演算法」**。系統整合了Raspberry Pi 4的電腦視覺推理與底層實體環境感測，透過Docker進行隔離編排。系統能精確交叉比對「火源存在」與「人員在場狀態」，動態識別是 **安全烹飪** 還是 **異常空燒**，並建置響應式網頁控制台實現低時延的跨網遠端安全遙測。

### 🌟 核心亮點 (Key Features)
* **多模態融合決策：** 揮別單一感測器誤報，結合語意標籤與物理量綜合判定。
* **邊緣運算微服務：** 全系統導入 Docker 容器化隔離，提升系統魯棒性與部署彈性。
* **AI 推理算力優化：** 採用輕量化 YOLO 模型，並加入 2.0 秒影像降頻保護機制，節省 75% 算力開銷。
* **跨平台響應式控制台：** 基於 Bootstrap 5 框架，手機/網頁雙端免 Port Forwarding 低時延同步遙測。

## 系統分層架構

全系統由下至上主要劃分為三大核心層級，軟硬體拓撲連線如下所示：

1.  **頂層：跨平台終端 (User Interface Layer)**
    * 以 Python Flask 作為 Web 伺服器，配合 Bootstrap 5 響應式網頁技術，實現跨平台（PC/手機版）即時前端渲染與數據折線圖（AJAX 非同步通信）。
2.  **中層：邊緣運算核心 (Docker Kernel Layer)**
    * **AI 視覺微服務：** 運行 YOLO 視覺推理引擎，提取環境空間狀態。
    * **數據接收微服務：** 監聽序列埠，解碼並清理底層感測數據。
    * **核心決策邏輯 ：** 匯整異質數據，執行階層式條件分流，控制警報狀態。
3.  **底層：實體感測硬體 (Hardware & Sensing Layer)**
    * 透過主控微處理器採集環境瓦斯與一氧化碳濃度，並對接實體蜂鳴器（Buzzer）實施秒級在地物理預警。

決策中樞（Logic）採取 **階層式四階段過濾機制**，確保危機響應之最高優先級：
開始]

├──► 數據並行輸入 (YOLO 影像標籤 + 瓦斯感測數值)
│
├──► [決策 1: 瓦斯異常?] ───(是: 讀值 > 700)───► 🚨 [瓦斯漏氣警報] ➔ 驅動蜂鳴器/雲端推播
│         │(否)
│         ▼
├──► [決策 2: 火災且無人?] ─(是: 有火且 None)──► 🍳 [異常空燒警報] ➔ 驅動蜂鳴器/雲端推播
│         │(否)
│         ▼
├──► [決策 3: 安全烹飪?] ───(是: 有火且有人)───► ✅ [系統安全狀態] ➔ 前端網頁安全顯示
│         │(否)
│         ▼
└──► [決策 4: 無危害?] ─────(是: 無火且正常)───► 🍃 [正常無危害] ➔ 維持綠色 SAFE 燈號

## 專題（二）預期演進：無線化與市售設備改造
為了提升系統的實務落地性與商品化價值，下一階段研發重點將全面升級：
1.  **硬體逆向工程改裝：** 拆解市售市占率最高之消防警報器，保留原廠消防認證之感測精準度與保底報警線路，透過**光耦隔離技術**並聯劫持訊號。
2.  **大幅降低系統誤報率** 連續多幀判定為火焰後才觸發告警，過濾偶發閃爍。
3.  **自信度門檻優化** 動態調校 YOLO 自信度閾值，確保辨識精度。
4. **感測基準漂移補償**實作 MQ-9 動態基準值校正，適應溫濕度變化。
5. **全平台一致的應用體驗**1.掃描 QR Code 即可加入主畫面圖示。
                        2.完整適配 iOS 與 Android 平台。
                        3.移除瀏覽器網址列，提供全螢幕 UI。
