##  專案簡介 
近年新聞頻傳因無人看火」或瓦斯管線老舊洩漏導致的住宅意外，造成嚴重財損與傷亡。傳統廚房安防設備常依賴單一物理感測器（如煙霧、瓦斯警報器），在實際烹飪情境中極易因高溫油煙或水蒸氣引發誤報。

本專題之核心創意為 **「多模態資料融合判定演算法」**。系統整合了Raspberry Pi 4的電腦視覺推理與底層實體環境感測，透過Docker進行隔離編排。系統能精確交叉比對「火源存在」與「人員在場狀態」，動態識別是 **安全烹飪** 還是 **異常空燒**，並建置響應式網頁控制台實現低時延的跨網遠端安全遙測。

###  核心亮點
* **多模態融合決策：** 揮別單一感測器誤報，結合語意標籤與物理量綜合判定。
* **邊緣運算微服務：** 全系統導入 Docker 容器化隔離，提升系統魯棒性與部署彈性。
* **AI 推理算力優化：** 採用輕量化 YOLO 模型，並加入 2.0 秒影像降頻保護機制，節省 75% 算力開銷。
* **跨平台響應式控制台：** 基於 Bootstrap 5 框架，手機/網頁雙端免 Port Forwarding 低時延同步遙測。

## 系統分層架構

全系統由下至上主要劃分為三大核心層級，軟硬體拓撲連線如下所示：

1.  **頂層：跨平台終端 (User Interface Layer)**
    * 以 Python Flask 作為 Web 伺服器，配合 Bootstrap 5 響應式網頁技術，實現跨平台（PC/手機版）即時前端渲染與數據折線圖（AJAX 非同步通信）。
2.  **中層：邊緣運算核心 (Docker Kernel Layer)**
    * **AI 視覺微服務：** 運行 YOLO 視覺推理引擎，提取環境空間狀態。
    * **數據接收微服務：** 監聽序列埠，解碼並清理底層感測數據。
    * **核心決策邏輯 ：** 匯整異質數據，執行階層式條件分流，控制警報狀態。
3.  **底層：實體感測硬體 (Hardware & Sensing Layer)**
    * 透過主控微處理器採集環境瓦斯與一氧化碳濃度，並對接實體蜂鳴器（Buzzer）實施秒級在地物理預警。

＃＃ 專案開發困境與彈性對策 

本團隊於專案開發與環境布建過程中，面臨多項硬體算力、模型精度、全端通訊及實務部署之瓶頸。以下為核心困境分析與對應之動態工程解決對策：
 困境一：邊緣端硬體算力不足與環境衝突

    技術困境： 團隊初期規劃於樹莓派）原生作業系統中直接布建 YOLO 推理引擎，然而面臨全域 Python 環境依賴庫與底層感測器序列埠）驅動程序發生嚴重相依性衝突，且連續偵測導致硬體核心過熱、系統頻繁卡死。

    工程對策： 團隊全面導入 Docker 容器化微服務架構。將視覺推理、序列數據解碼與 Flask Web Server 進行模組化解耦與環境隔離。此舉不僅徹底消除環境衝突，更透過 Docker 限制單一容器之 CPU 算力上限，成功讓系統在低算力邊緣端穩定運行。

困境二：模型信心度不足與評審現場誤判風險

    技術困境： 模型訓練完成初期，因預設環境背景雜訊較多，導致特定角度下視覺辨識的信心度偏低。在畢業專題成果報告準備期間，甚至偶發性出現標籤誤判（如將暗影誤判為火源、或漏判人員標籤）的極端情境，恐影響多模態決策之準確性。

    工程對策：

        資料集擴充與再訓練： 團隊緊急導入更多元的開源資料集（如 Fire Smoke and Human Detector），針對低光源、高油煙等極端廚房情境實施模型微調與強化訓練，大幅提升模型泛化能力。

        多模態交叉印證機制： 為消除使用者對單一相機鏡頭誤判的疑慮，我們在前端 Bootstrap 網頁儀表板上，創新結合了實體感測器（MQ-9）的即時數據折線圖。當視覺端發生瞬間掉幀或誤判時，使用者可透過高動態更新的氣體曲線進行交叉核對，用「影像＋物理數據」雙重防線確保情境判定的真實性。

 困境三：內網穿透工具（Ngrok）資源限制與資安瓶頸

    技術困境： 系統初期使用 Ngrok 實施內網穿透，並動態生成網頁 QR Code 供遠端持有者掃描加入檢視。然而，Ngrok 免費版具備連線次數與通道數上限。在專題報告前夕，因高頻率的測試與展示，導致 QR Code 存取次數在評審現場直接達到上限、宣告封鎖，無法透過動態機制提供網址。

    工程對策： 團隊於報告現場立即啟動應變方案，將邊緣閘道端（樹莓派）與前端檢視終端（手機/PC）統一接入同一個本地網域基地台。改由直接布建本地伺服器監聽，讓所有評審委員透過區域網路的固定 IP 與網址直接進入系統網頁進行遙測。此舉不仅成功破解穿透工具的次數限制，亦縮短了資料在外網轉發的時延。

## 專題（二）預期演進：無線化與市售設備改造
為了提升系統的實務落地性與商品化價值，下一階段研發重點將全面升級：
1.  **硬體逆向工程改裝：** 拆解市售市占率最高之消防警報器，保留原廠消防認證之感測精準度與保底報警線路，透過**光耦隔離技術**並聯劫持訊號。
2.  **大幅降低系統誤報率** 連續多幀判定為火焰後才觸發告警，過濾偶發閃爍。
3.  **自信度門檻優化** 動態調校 YOLO 自信度閾值，確保辨識精度。
4. **感測基準漂移補償**實作 MQ-9 動態基準值校正，適應溫濕度變化。
5. **全平台一致的應用體驗**1.掃描 QR Code 即可加入主畫面圖示。2.完整適配 iOS 與 Android 平台。3.移除瀏覽器網址列，提供全螢幕 UI。
